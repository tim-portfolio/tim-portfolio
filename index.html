Featured Projects

SRR — Agentic Case Processing System

Hong Kong · 2024–2025
Lead AI Architect & Full-Stack Developer (99.4% contribution)

Problem

Hong Kong government slope repair applications required 40+ minutes of manual processing per case, with inconsistent quality across officers and high operational cost (~HK$8.52M/year).

Data

Multi-format documents: PDF slope reports, government regulation PDFs, historical GIS records, and citizen application forms — processed via OCR and structured extraction.

Approach

Designed a 7-layer Agent architecture with 15 atomic capabilities. Core innovations: LLM-as-Judge quality gate (semantic + structural + hallucination checks), self-repair loops, multi-source RAG with pgvector, and code-driven orchestration with LLM-only for decision nodes.

Outcome

Reduced processing time by 75% (40 min → 10 min). Achieved 99.95% data consistency. Projected annual cost savings of HK$6.39M. System handles 15,000+ cases/year with sub-minute response time.

Tech Stack

Python · LangChain · OpenAI GPT-4 · Google Gemini · PostgreSQL + pgvector · FastAPI · Docker · GCP Cloud Run

LingO — Multi-Agent University Consultation System

Lingnan University · 2024–2025
System Architect & Lead Developer

Problem

University students, faculty, and administrators struggled with dispersed policy documents across 30+ departments, leading to 70%+ retrieval failure rates and inconsistent information delivery.

Data

Ingested 200+ policy documents, academic regulations, administrative guidelines, and FAQ corpora across three user personas (student, faculty, admin).

Approach

Outcome

Policy Q&A accuracy >95%. Average retrieval time reduced by >70%. System response within 10 seconds. Demonstrated at Lingnan Innovation Showcase.

Tech Stack

Python · LangChain · OpenAI API · PostgreSQL + pgvector · FastAPI · Streamlit · Docker · GCP
